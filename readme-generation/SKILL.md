# 技能规范说明（Skills Specification）

本文件用于定义 **生成项目 README.md 的技能规范（Skills）**。

该技能集用于约束和指导 AI 在生成 README 时的结构、内容边界与表达重点。

本规范遵循 Codex 官方提出的 **“Skill = 面向任务的可执行能力单元”** 设计理念，

目标是生成一份：

- **只读型**

- **面向使用者**

- **强调编译 / 部署 / 使用 / 架构理解**

- **不过度暴露实现细节**

的工程级 README 文档。

---

## 🎯 Skill 01：项目概述生成

**目标**

生成对项目的高层次概述，作为 README 的入口说明。

**能力要求**

- 用简洁语言回答“这个项目是做什么的”

- 不依赖代码实现细节

- 不使用宣传性、营销性描述

**对应 README 章节**

- README 顶部简介

- 📌 项目概述（Overview）

---

## 🏗️ Skill 02：系统整体架构表达

**目标**

以结构化方式描述系统的整体架构与组件边界。

**能力要求**

- 明确区分各个模块等运行侧

- 能使用 ASCII 字符画表达组件关系

- 强调“组件职责”而非“实现方式”

**约束**

- 不包含任何代码

- 不暴露协议字段、数据结构或内部算法

**对应 README 章节**

- 🏗️ 整体架构（Architecture）

- 🧭 架构总览（ASCII）

---

## 🔄 Skill 03：通信与运行模型说明（如果有则生成反之则不生成）

**目标**

解释系统在运行时是如何协同工作的，如果有通信则章节名为 通信模型 反之章节名为运行模型。

**能力要求**

- 描述组件如何启动、通信与配合

- 使用流程性、概念性语言，而非底层实现细节

**对应 README 章节**

- 🔄 通信与运行模型（Runtime Model）

- 🚀 部署（Deploy）

---

## 🧰 Skill 04：环境与依赖条件描述

**目标**

明确工具正常运行所需的环境条件。

**能力要求**

- 区分编译依赖与运行依赖

- 面向“使用者如何准备环境”

- 避免与具体发行版或精确版本强绑定

**对应 README 章节**

- 🧰 环境与依赖（Requirements）

---

## 🛠️ Skill 05：构建流程说明

**目标**

说明如何从源码构建出可运行的产物。

**能力要求**

- 描述构建的整体步骤与顺序

- 如果使用cmake 或者其他编译工具 则 简单几行 简述 构建编译过程即可。

- 面向使用者，而非构建系统维护者

**对应 README 章节**

- 🛠️ 编译（Build）

---

## 🚀 Skill 06：部署

**目标**

说明构建完成后的运行与部署方式。

**能力要求**

- 解释程序的运行形态

- 描述运行所处的环境与上下文

- 使用中性、技术性的表述

**对应 README 章节**

- 🚀 部署（Deploy）

---

## 🎮 Skill 07：使用模型构建

**目标**

为工具建立清晰、稳定、可理解的使用模型。

**能力要求**

- 从“如何使用工具”而非“如何实现工具”出发

- 使用指南必须包含以下子结构：

- 一句话概述 xx工具提供了xx功能

- 基本语法 工具 <参数> 这种形式

- 通用选项 / 程序选项

- 模块使用说明 工具 功能模块参数 <参数>

- 查看帮助 help展示

- 典型使用场景 具体参数 和 具体返回结果

**对应 README 章节**

- 🎮 使用指南（Usage Guide）

---

## 🧱 Skill 08：模块化能力说明

**目标**

说明工具的功能是如何被拆分和组织的。

**能力要求**

- 按“能力/用途”而非“源文件或代码结构”描述模块

- 允许部分列举，不要求完整穷举

- 不描述模块内部实现逻辑

**对应 README 章节**

- 使用指南 → 模块使用说明

---

## 💡 Skill 09：基于使用场景的说明能力

**目标**

通过典型使用场景帮助读者理解工具用途。

**能力要求**

- 以“问题或目标”为出发点

- 将多个功能能力组合到一个场景中

- 不依赖真实生产环境或敏感数据

**对应 README 章节**

- 使用指南 → 典型使用场景

---

## 🗂️ Skill 10：文档结构与边界感知

**目标**

保证 README 结构清晰、稳定、长期不过期。

**能力要求**

- 各章节职责清晰、互不重叠

- 不引入协作、贡献、CI、测试等非目标内容

- 明确 README 的定位是“只读型使用文档”

**适用范围**

- README 全文生成过程

---

## ✅ 使用约定

在生成或更新 README 时，应遵循以下原则：

1. 本 SKILL.md 作为能力与内容边界的唯一约束来源

2. README 的每个主要章节应至少对应一个 Skill

3. 当缺少必要信息时，应保留明确占位说明，而不是猜测或虚构

4. 每个章节开始部分都必须有与之对应的emoij
